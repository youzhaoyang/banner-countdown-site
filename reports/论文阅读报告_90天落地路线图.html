目标：把“长上下文、Agent稳定性、异构模型降本”三条结论，转成90天内可上线、可量化的实施计划。

总目标与成功标准

目标 A

性能

长文档/长代码任务正确率显著提升，减少上下文截断导致的遗漏。

目标 B

稳定

Agent 连续运行失败率降低，可回滚、可恢复、可审计。

目标 C

成本

建立LLM/SLM路由，降低高频任务token成本并保持体验。

90天 KPI（建议）

  • 长上下文任务成功率提升 ≥ 20%。
  • Agent 长周期任务中断率下降 ≥ 50%。
  • 高频流程任务单位成本下降 30%-60%。
  • 关键链路（OCR/抽取）人工复核量下降 ≥ 30%。

分阶段计划（总-分-总执行）

阶段时间核心动作产出物验收指标
P0 基线建设 第1-2周 梳理任务分层、建立评测集、接入日志与成本看板 任务分类表、评测脚本、观测仪表盘 100%关键流程可观测
P0 稳定性护栏 第3-4周 为Agent加状态快照、异常回滚、目标重对齐 护栏中间件、恢复策略手册 连续运行失败率下降30%+
P1 异构路由 第5-7周 上线LLM/SLM路由（复杂任务→LLM，重复任务→SLM） 路由策略、灰度开关、A/B结果 成本下降20%+且质量不降
P1 长上下文增强 第8-10周 长文档/代码链路接入稀疏或混合注意力后端并压测 性能对比报告、容量建议 长任务正确率提升15%+
P2 业务扩面 第11-13周 从1-2个试点流程扩展到多部门流程并固化SOP 上线清单、运维手册、治理规范 稳定运行2周+，复盘闭环

关键设计（对应论文结论）

1) 路由策略模板（SLM/LLM）

  • 规则任务（改写、模板填充、FAQ）默认走SLM。
  • 跨文档归因、多步推理、低容错任务走LLM。
  • 置信度低或冲突结果自动升级到LLM复核。

作用示例：客服工单摘要先SLM批处理，遇到法律条款争议自动升级LLM，兼顾吞吐与准确率。

2) Agent护栏模板（长期稳定）

  • 每N步写入状态快照：目标、当前资源、下一步计划。
  • 出现“连续无效动作”触发回滚到最近稳定点。
  • 每轮执行前后做目标一致性检查（防偏航）。

作用示例：自动运营Agent出现重复下单趋势时，系统自动回滚并切入人工审批模式。

3) 验证层模板（Verifier）

  • 保留原OCR/规则模型，只新增LLM验证层进行冲突仲裁。
  • 关键字段（名称、地址、电话）输出置信度与解释。
  • 低置信度样本进入人工复核池，反哺训练集。

作用示例：门店识别中“0/O、1/I”混淆通过LLM语义校验修正，减少误收录。

风险与应对

风险触发信号应对策略

最终交付清单

  1. 评测与观测：任务评测集、稳定性压测脚本、成本看板。
  2. 平台能力：LLM/SLM路由器、Agent护栏中间件、Verifier服务。
  3. 治理文档：上线SOP、回滚预案、模型选择准则。
